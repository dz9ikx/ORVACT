# Contributing to ORVACT

## Principles
1. Openness: All contributions must be compatible with CC-BY-SA 4.0 / CERN-OHL-S-2.0.
2. Safety: Proposals must include risk assessment for high-speed rotation and hazardous materials.
3. Reproducibility: Experimental data must include methodology and raw logs.

## How to Contribute
1. Fork the repository.
2. Create a feature branch (e.g., `feature/cfd-analysis`).
3. Make changes and test locally.
4. Submit a Pull Request with:
   - Clear description of changes.
   - Reference to related Issues.
   - Test results or simulation data (if applicable).

## Code and Design Standards
- CAD: Use STEP format for interchange. Include source files (Fusion360, FreeCAD).
- Firmware: C/C++, Arduino framework. Comment critical sections.
- Documentation: Markdown. Use SI units.

## Prohibited Contributions
- Patent-encumbered designs or code.
- Proprietary binary blobs without source.
- Modifications that violate the ORVACT Open Humanity License terms.

## Communication
- Use GitHub Issues for bugs and feature requests.
- Use GitHub Discussions for theoretical questions.

## License Agreement
By submitting a contribution, you agree to license it under the project's existing licenses (CC-BY-SA 4.0 for docs, CERN-OHL-S-2.0 for hardware, MIT for software) and the ORVACT Open Humanity License v1.0.
